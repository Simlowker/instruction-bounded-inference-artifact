## Conclusion

The 29-vs-10 / 2.9× claim is **fully reproducible from current public source** on a local dfx replica. The result is robust to:

- Two months of source drift on the Comoto side (initial commit → 8cda13b)
- One minor version upgrade on the onicai side (v0.9.0 → v0.10.1)
- Toolchain drift (clang 18 → clang 21)
- Local environment vs ICP mainnet (same 40 B instruction cap, dfx pocket-ic enforces identically for this metric)

The original 2026-04-09 binaries are not byte-reproducible (working-tree state was never committed; see paper §6 note and README.md). Byte-exact rebuild was not pursued because functional reproducibility — the actual claim — is established.
