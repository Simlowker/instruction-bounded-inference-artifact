## Claim Status Legend

- `SOLID`: reproduced or analytically closed, with artifact traceability in place.
- `PARTIAL`: supported, but still limited by repetitions, sample breadth, or packaging gaps.
- `WEAK`: suggestive only; the paper should keep cautious wording.
- `RETIRED`: claim deliberately removed or downgraded from the current draft.

## Row Status Convention

The citation-facing tables now use one stable Workstream C convention in `models.csv` and `artifact/data/onchain/*.csv`:

```text
runs=<...>; quality=<...>; env=<...>; build=<...>; role=<...>
```

The keys always appear in that order. Composite values use `+` within one field.

- `runs` distinguishes `repeated` from `single_run`.
- `quality` distinguishes `verified`, `limited`, `invalid`, and `not_checked`.
- `env` distinguishes `local`, `icp_mainnet`, `ssn_mainnet`, and `native_only`.
- `build` distinguishes `current`, `historical`, `build_sensitive`, `build_specific`, and `unknown`.
- `role` distinguishes `calibration`, `supporting_measurement`, `network_validation`, `systems_demo`, `negative_result`, `baseline`, and `native_only`.

This is the mechanism by which the reviewer can tell, row by row, what feeds the law, what is a network validation, what is a systems demonstration, and what is a negative result.

## Current Gaps Before Freeze

Both drafts are readable and reviewable, but these are still real limits:

- The scaling-law calibration (C2) is only partially repetition-backed.
- The embedding section (C10) remains a system demonstration, not a benchmark-grade evaluation.
- Mainnet validation (C4) is narrow: one flagship pair, not a broad repeated network sweep.
- Matmul dominance (C5) is measured clearly, but on a narrow profiling base (one configuration).
- Several public-network rows remain explicitly `build=build_specific`; disclosed, not resolved.
- Ternary floor (C12) is anchored on one model family and one quantization format.
- Multi-call IO fits (C15, C16) cover two models (Transformer + one hybrid SSM); pure-Mamba baseline and broader cross-architecture sweep are not yet measured.
- Load-ceiling characterization (C17, C18) is operational-observation depth, not a systematic sweep across size, format, and ICP subnet.
